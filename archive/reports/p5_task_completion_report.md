## 📊 **系统架构设计**

### **模块化架构**
```
quant_system/
├── trading/           # 实时交易系统
│   ├── core.py       # 交易引擎核心
│   ├── models.py     # 数据模型（订单、账户等）
│   └── strategies/   # 交易策略
├── risk_management/  # 风险管理系统
│   ├── core.py       # 风险管理核心
│   ├── models.py     # 风险模型（指标、告警等）
│   └── analysis/     # 风险分析工具
├── reporting/        # 报告生成系统
│   ├── core.py       # 报告生成核心
│   ├── templates/    # 报告模板
│   └── charts/       # 图表生成
└── p5_integration.py # P5集成模块
```

### **数据流转**
```
实时交易数据 → 交易引擎 → 持仓更新 → 风险分析 → 风险监控 → 报告生成 → 可视化输出
     ↓              ↓          ↓          ↓          ↓          ↓          ↓
市场数据接口     订单处理     仓位计算     指标计算     告警检查     模板渲染     用户界面
```

### **API端点**
```
GET    /api/p5/status                    # 系统状态
GET    /api/p5/accounts                  # 所有账户
GET    /api/p5/accounts/<account_id>     # 账户详情
POST   /api/p5/accounts                  # 创建账户
POST   /api/p5/orders                    # 下达订单
POST   /api/p5/market_data               # 更新市场数据
GET    /api/p5/risk/<account_id>         # 风险分析
POST   /api/p5/stress_test/<account_id>  # 压力测试
POST   /api/p5/reports/daily/<account_id># 生成日报
POST   /api/p5/backtest                  # 运行回测
```
