# 每周操作流程 - 量化策略监控

## 📅 每周日盘后工作流

### 第一周（建立观察池）

```powershell
# 1. 清缓存 (必要 - 确保数据最新)
Remove-Item -Recurse -Force .\data\*

# 2. 全市场扫描 (15-30 分钟)
python main.py scan --show-near-miss

# 3. 保存得分≥3 分的股票到观察池
python watchlist.py save --min-score 3
```

**预期输出：**
- 全市场扫描：6494 只股票
- 观察池：50-100 只（得分 3-7 分）
- 触发：0 只（正常状态）

---

### 第二周及以后（追踪观察池）

```powershell
# 4. 清缓存
Remove-Item -Recurse -Force .\data\*

# 5. 只扫描观察池 (快 10 倍，1-3 分钟)
python watchlist.py update

# 6. 对比两周变化，看谁得分上升
python watchlist.py diff
```

**预期输出：**
- 观察池更新：50-100 只
- 得分变化：哪些从 3 分→4 分→5+ 分
- 新进入观察池的股票
- 被剔除的股票

---

## 🎯 核心逻辑

### 为什么要这样做？

1. **避免盲目扫描**：每周扫 6494 只效率低，观察池只需追踪 50-100 只
2. **提前发现信号**：股票从 3 分→4 分→5+ 分需要时间，有足够建仓窗口
3. **聚焦边际变化**：关注"接近触发"的票，而不是已经触发或完全不符合的

### 观察池运作机制

```
Week 1: 全市场扫描 → 发现 50 只 3-4 分股票 → 存入观察池
Week 2: 只扫这 50 只 → 发现 3 只升到 5+ 分 → 准备建仓
Week 3: 继续追踪 → 看是否有新股票进入观察池
```

## 📊 关键指标解读

### 得分分布示例

```
score
0    4100  ← L0 都没通过 (数据不足/基本面差)
1    2288  ← 通过 L0，倒在 L1 (回撤不够)
2      91  ← 通过 L1，倒在 L2 (无低位涨停)
3      12  ← 通过 L2，倒在 L3 (无缺口)
4       3  ← 通过 L3，倒在 L4 (连阳不足) ← **重点追踪**
5       0  ← 通过 L4，倒在 L5 (量比不足) ← **准备建仓**
6+      0  ← 完全触发 ← **买入信号**
```

### 各层失败次数（瓶颈分析）

```
L1: 2448 次 ← 最严苛瓶颈 (距 500 日高点回撤不足)
L2: 2288 次 ← 次严瓶颈 (近 60 日无低位涨停)
L0: 1652 次 ← 数据不足或基本面过滤
L3:   91 次
L4:   12 次 ← **重点关注** (接近触发)
L5:    3 次
```

## 🛠️ watchlist.py 命令详解

### `save` - 保存观察池

```powershell
# 保存得分≥3 分的所有股票
python watchlist.py save --min-score 3

# 保存得分≥4 分的股票（更严格）
python watchlist.py save --min-score 4
```

**输出文件：** `results/watchlist_YYYYMMDD.csv`

---

### `update` - 更新观察池

```powershell
# 读取上次保存的观察池，只扫描这些股票
python watchlist.py update
```

**优势：** 速度快 10 倍（从 15 分钟→1-3 分钟）

---

### `diff` - 对比变化

```powershell
# 对比本周 vs 上周观察池
python watchlist.py diff
```

**输出：**
- 得分上升的股票（3→4→5+）
- 新进入观察池的股票
- 被剔除的股票（得分下降）
- 触发信号的股票（6+ 分）

## ⚠️ 注意事项

### 必须清缓存的场景

- ✅ 每周日盘后例行扫描
- ✅ 策略代码修改后
- ✅ 发现数据异常时

### 可以不清缓存的场景

- ✅ 只跑观察池更新（watchlist.py update）
- ✅ 回测历史数据（用 cache 加速）

### 最佳实践

1. **固定时间**：每周日 15:00 收盘后执行
2. **记录结果**：把每次观察池变化记录到 `memory/watchlog_YYYYMMDD.md`
3. **持续追踪**：至少追踪 4 周，看完整周期
4. **不要频繁调参**：策略已处在"等待市场"状态，阈值合理

## 🎓 常见问题

### Q: 为什么总是 0 只触发？

**A:** 正常！这是"等待市场"状态。策略设计就是低频高胜率，一个月可能只有 0-3 只触发。

### Q: 观察池多少只合适？

**A:** 50-100 只。太少说明策略太严，太多说明太松。

### Q: 多久调一次参数？

**A:** 至少 3 个月。每周调参数是过度拟合。

### Q: 什么时候该放弃策略？

**A:** 连续 3 个月 0 触发，或触发的票全部亏损。
