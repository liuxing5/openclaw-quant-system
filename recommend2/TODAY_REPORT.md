## ✅ 已修复问题

### 1. 股票代码格式问题 ✅ 已修复

**问题：** CSV 读取时 `symbol` 列被当成数字，导致前导零丢失（`002428` → `2428`）

**修复方案：**
```python
# watchlist.py 中所有 pd.read_csv() 都添加 dtype={"symbol": str}
base = pd.read_csv(base_file, dtype={"symbol": str})
df = pd.read_csv(scan_file, dtype={"symbol": str})

# 保存时确保 symbol 列是字符串格式
watch["symbol"] = watch["symbol"].astype(str)
```

**影响范围：**
- `cmd_save()` - 主升前夜观察池保存
- `cmd_update()` - 主升前夜观察池更新
- `cmd_save_dragon()` - 龙头观察池保存
- `cmd_update_dragon()` - 龙头观察池更新

**验证结果：**
- ✅ 云南锗业（002428）数据获取成功
- ✅ 4 只观察池股票全部正常扫描
- ✅ 无 baostock 代码格式错误

### 2. 板块同步度注入 ✅ 已优化

**优化：** `update-dragon` 命令现在支持板块同步度预计算

**修复前：** 直接调用 `scan_universe_dragon()`，无板块同步度数据
**修复后：** 先预计算板块同步度，再注入到扫描函数

**代码变更：**
```python
# 预计算板块同步度（用观察池数据）
print("预计算板块同步度 (L4)...")
sector_sync_map = precompute_sector_sync(
    symbols, loader=loader, days=30, end_date=datetime.now(TZ).strftime("%Y%m%d")
)
print(f"板块同步度：涵盖 {len(sector_sync_map)} 个交易日")

# 正式扫描（注入板块同步度）
result = scan_universe_dragon(
    symbols, cfg=cfg, loader=loader,
    sector_sync=sector_sync_map, eval_date=datetime.now(TZ).strftime("%Y-%m-%d")
)
```

**效果：**
- ✅ L4 层逻辑正常工作
- ✅ 板块同步度计算快速（观察池仅 4 只股票）
